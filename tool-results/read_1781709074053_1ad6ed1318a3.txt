   414→## 💡 MY SUGGESTIONS & CAVEATS
   415→
   416→### 1. Server Component Migration — HYBRID Approach
   417→**Original:** Full Server Component migration untuk semua pages
   418→**Suggestion:** App kita guna Zustand SPA routing. Full migration akan break:
   419→- Instant page transitions (Framer Motion)
   420→- Command palette (needs client state)
   421→- Keyboard navigation (needs client event listeners)
   422→- Live notifications (needs client WebSocket)
   423→
   424→**Recommendation:** Keep pages as Client Components. Hanya jadikan LandingPage dan static content sebagai Server Components.
   425→
   426→### 2. External AI Services — Use z-ai-web-dev-sdk
   427→**Original:** FAL.ai (images), OpenAI TTS, Browserbase
   428→**Suggestion:** Kita sudah ada `z-ai-web-dev-sdk` yang support:
   429→- LLM chat (HERMES)
   430→- Image generation (AI Thumbnails) ✅
   431→- TTS (AI Voiceover) ✅
   432→- Web search
   433→- Web page reading
   434→- Video understanding
   435→
   436→**Recommendation:** Guna z-ai-web-dev-sdk untuk semua AI capabilities. Hanya tambah external service bila z-ai-web-dev-sdk tak support.
   437→
   438→### 3. PostgreSQL Migration — DEFER
   439→**Original:** Migrate SQLite → PostgreSQL
   440→**Suggestion:** SQLite cukup untuk:
   441→- Single-user demo
   442→- Development environment
   443→- Small team (< 10 users)
   444→
   445→**Recommendation:** Defer ke Phase 6+ atau bila:
   446→- Multi-user concurrent aktif
   447→- Data volume > 100K records
   448→- Multi-instance deployment diperlukan
   449→
   450→### 4. Browser Automation/Scraping — AVOID
   451→**Original:** Stealth browser layer dengan Playwright, proxy rotation, spoofed navigator
   452→**Suggestion:** Shopee anti-bot detection sangat aggressive. Scraping boleh sebabkan:
   453→- IP ban
   454→- Account suspension
   455→- Legal/ToS violation
   456→
   457→**Recommendation:** Guna **official Shopee Affiliate API** sahaja (already ada `src/lib/shopee/affiliate-api.ts`). Fallback ke demo data bila API unavailable. Jangan scrape.
   458→
   459→### 5. Messaging Gateway — Telegram First
   460→**Original:** Telegram + WhatsApp + Discord + Slack + Email
   461→**Suggestion:** Untuk Malaysian affiliate market:
   462→- **Telegram** = priority #1 (paling popular di MY)
   463→- WhatsApp = perlu Meta Business approval (complex)
   464→- Discord = bukan primary market untuk MY affiliates
   465→- Slack = enterprise, tak relevant
   466→- Email = guna existing notification system
   467→
   468→**Recommendation:** Implement Telegram bot sahaja dulu (Phase 6a). Defer yang lain.
   469→
   470→### 6. Computer-Use Agent — DEFER
   471→**Original:** Split-screen VLA loop dengan Playwright + GPT-4o Vision
   472→**Suggestion:** Ini feature paling complex (80h+ effort). Untuk sekarang, text-based agent (HERMES chat) dah cukup powerful.
   473→
   474→**Recommendation:** Defer ke Phase 6+ atau bila core Hermes v2 (Memory + Skills + Cron + Delegation) dah stable.
   475→
   476→### 7. Sentry — DEFER
   477→**Original:** Install Sentry untuk error monitoring
   478→**Suggestion:** Sentry free tier terhad (5K errors/month). Kita sudah ada:
   479→- Structured logger (`src/lib/logger.ts`)
   480→- Error boundaries (Global + per-page)
   481→- `handleApiError()` helper
   482→
   483→**Recommendation:** Defer Sentry ke bila production traffic meningkat. Untuk sekarang, logger + error boundaries cukup.
   484→
   485→### 8. Page Component Refactoring — PRIORITIZE
   486→**Original:** Refactor page components > 1000 baris
   487→**Suggestion:** Local codebase kita tak ada page > 500 baris (most are 200-400). Tapi kalau future pages grow besar, refactor ke sub-components.
   488→
   489→**Recommendation:** Tambah rule di CONTRIBUTING.md: "Page components should be < 500 lines. Extract sub-components untuk complex sections."
